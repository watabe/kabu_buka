**`ax.clear()` 問題の詳細:**
`plot_intraday_live()` の `animate()` 内で `ax1.clear()` を呼ぶと、`axes[0]` に追加したすべてのアーティスト（オーバーレイのライン・テキスト含む）が消える。そのため、`mpf.plot(plot_df_new, ax=ax1, ...)` の直後に毎フレーム `draw_prediction(ax1, result)` を再呼び出しする必要がある。

#### パターン検出ロジック（ルールベース）

実装難易度と実用性のバランスから以下4パターンを対象とする:

| パターン名 | 検出条件の概要 | 方向 |
|---|---|---|
| ダブルトップ | Close の直近N本で2つの極大値が近似価格で現れ、その間に極小値がある | down |
| ダブルボトム | Close の直近N本で2つの極小値が近似価格で現れ、その間に極大値がある | up |
| ヘッドアンドショルダー | 極大値3つ（中央が最高）かつ両肩の高さが近似 | down |
| 逆ヘッドアンドショルダー | 極小値3つ（中央が最低）かつ両肩の深さが近似 | up |

検出には `scipy.signal.find_peaks` または単純な rolling window での極値検出を使用する。データ本数が 30 本未満の場合は検出をスキップして `None` を返す。

#### 予測ライン生成

- 最後の Close 価格から direction に応じて N 本（例: 10 本）先まで線形外挿
- 将来のタイムスタンプは最終行の freq（1min）を使って `pd.date_range` で生成

```python
def _extrapolate_index(df: pd.DataFrame, n_steps: int = 10) -> pd.DatetimeIndex:
    last_ts = df.index[-1]
    freq = pd.tseries.frequencies.to_offset("1min")
    return pd.date_range(start=last_ts + freq, periods=n_steps, freq=freq, tz=last_ts.tz)
```

## Risks/Constraints Discovered

1. **`plot_intraday()` の `axes` 未取得**: 現状 `fig, _ = mpf.plot(...)` で axes を捨てているため、`fig, axes = mpf.plot(...)` に変更が必要。既存テストの `mock_plot.return_value = (MagicMock(), MagicMock())` で `_` だったものが `axes` になるが、`plot_intraday_live()` と同様のタプルアンパック変更のみで対応可
2. **scipy 依存の追加**: `scipy.signal.find_peaks` を使う場合、`requirements.txt` / `pyproject.toml` への追加が必要。回避策として `numpy` の `argrelextrema` か自前のスライディングウィンドウ実装を使えば scipy なしで実現できる
3. **`animate()` でのオーバーレイ再描画**: `ax1.clear()` 後に毎フレーム `detect_pattern()` と `draw_prediction()` を呼ぶため、処理コストが増加する。`detect_pattern()` はネットワーク呼び出しなしの純粋計算なので許容範囲内
4. **`--predict` なしの場合の後退テスト**: シグネチャに `predict=False` を追加するが、既存テストはすべてデフォルト引数で呼ぶため変更不要
5. **パターン表示位置**: Row 3 の meta テキストは `_meta_txt.set_text()` で管理されているが、パターン名を meta に追加すると `_build_info_parts()` の `meta` フィールドの変更が必要。代替として `ax.set_title()` や `ax.annotate()` でチャート内に表示する方が関心の分離として好ましい

## Open Questions

1. `scipy` を依存に追加してよいか、それとも自前の極値検出で完結させるか → ドキュメントでは scipy なしの自前実装を採用する
2. パターン名の表示位置: Row 3 meta テキストに追記するか、`axes[0]` 上に `ax.annotate()` で直接描画するか → チャート内 `ax.annotate()` のほうが視覚的に位置が明確。ただし `ax.clear()` で消えるため animate 内で再描画が必要
3. 予測外挿の本数（`n_steps`）: 10 本固定か、既存データ本数の一定割合（例: 10%）か → 実装はシンプルに 10 本固定とする
4. 検出できなかった場合にチャート上に「パターン未検出」と表示するか、何も表示しないか → チャート左下に小さく表示する（情報パネルの Row 3 に付加する）
5. `--predict` と `--interval` を併用した場合（live モード + predict）: `animate()` 内でも毎フレーム予測を更新するか → 毎フレーム更新する（データが新しくなるたびに再検出）
