## Findings Summary

### 1. yfinance intraday データ取得

`yf.Ticker("7203.T").history(period="1d", interval="1m")` で東証銘柄の1日分・1分足データを取得できる。

```python
import yfinance as yf
df = yf.Ticker("7203.T").history(period="1d", interval="1m")
# df.columns: Open, High, Low, Close, Volume, Dividends, Stock Splits
# df.index: DatetimeIndex (tz=Asia/Tokyo)
```

- **対応インターバル（period="1d" との組み合わせ）:** `1m`, `2m`, `5m`, `15m`, `30m`, `60m`, `1h`
- **1m データの制約:** 直近 7 日分のみ取得可能
- **タイムゾーン:** インデックスは `Asia/Tokyo`（JST, UTC+9）で自動返却される
- **東証取引時間:** 09:00〜11:30、12:30〜15:30 JST（ランチブレークあり）
- **データ遅延:** 無料プランでは 15〜20 分遅延の可能性あり

### 2. チャートライブラリ比較

| 項目 | matplotlib (素) | mplfinance | plotly |
|---|---|---|---|
| ローソク足サポート | 手動実装が必要 | ネイティブ（1行） | ネイティブ（1行） |
| ネイティブウィンドウ | Yes | Yes | No（ブラウザ） |
| yfinance DataFrame 直接利用 | Yes | Yes | Yes |
| macOS GUI 表示 | 要バックエンド設定 | 自動（問題少ない） | ブラウザタブ |
| コード量 | 多い | 少ない | 少ない |
| インストール | 標準（多くの場合） | `pip install mplfinance` | `pip install plotly` |

**推奨: `mplfinance`** — ローソク足・出来高サブプロットを1行で表示でき、yfinance の DataFrame と直接互換。ブラウザ不要のネイティブウィンドウで動作する。

```python
import mplfinance as mpf
mpf.plot(df[["Open","High","Low","Close","Volume"]], type="candle",
         style="yahoo", title="7203.T", volume=True)
```

### 3. macOS バックエンドの注意点

- デフォルトは `MacOSX`（Cocoa）バックエンド。通常は動作するが、Homebrew Python やフレームワーク外 Python では失敗することがある。
- フォールバックとして `TkAgg` が有効（`brew install python-tk` または `conda install tk` が必要な場合あり）。
- `matplotlib.use(...)` は `import matplotlib.pyplot` より**前**に呼ぶ必要がある。
- 環境変数 `MPLBACKEND=TkAgg python main.py 7203` で外部から指定するのが最も安全。
- `mplfinance` は `matplotlib` の通常のバックエンド検出に乗るため、素の `matplotlib` より問題が出にくい。

### 4. argparse との統合パターン

既知の問題（matplotlib Issue #1986）: `matplotlib.pyplot` を先にインポートすると argparse が `sys.argv` を正しく処理できない場合がある。

**正しい順序:**
1. `import argparse` してから `parse_args()` を実行
2. その後で `import mplfinance` / `import matplotlib` する

現在の `main.py` は argparse 後に `src.fetcher` / `src.display` をインポートしているため、`src/chart.py` で mplfinance をモジュールレベルでインポートしなければ問題は生じない（関数内インポートを使えばさらに安全）。

## Risks / Constraints Discovered

- **1m データは直近 7 日のみ:** `period="1d"` の組み合わせでは実質的に本日〜7日前までに限定。週末・祝日を跨ぐと当日データが空になりうる。
- **取引時間外のデータなし:** 市場が閉まっている時間に実行すると `df` が空 DataFrame になる。空チェックが必須。
- **15〜20分の遅延:** 無料プランはリアルタイムではない。
- **macOS GUI バックエンド問題:** Homebrew Python / venv 環境では `MacOSX` バックエンドが使えず、`TkAgg` へのフォールバックが必要になる場合がある。`python-tk` の追加インストールが必要になりうる。
- **mplfinance の追加依存:** `requirements.txt` に `mplfinance>=0.12.10` を追加する必要がある。
- **ヘッドレス環境（CI等）:** GUIバックエンドがない環境では `plt.show()` がエラー。CI ではバックエンドを `Agg`（非表示・ファイル保存のみ）に切り替える必要がある。
