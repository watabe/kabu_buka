## Target price calculation per pattern

各パターンの理論的な目標価格の計算方法を整理する。

### ネックライン系パターン（H&S、ダブル、トリプル）

```
ダブルトップ:
  neckline = close[between_minima の最安値インデックス]
  target   = neckline - (peak_avg - neckline)   # ピーク平均からネックラインまでの高さ分、ネックラインから下

ダブルボトム:
  neckline = close[between_maxima の最高値インデックス]
  target   = neckline + (neckline - trough_avg)

ヘッドアンドショルダー:
  neckline = (close[左肩右の谷] + close[頭右の谷]) / 2  ← 肩の間の谷が2つある場合
             または minima の中から l_idx < m < h_idx と h_idx < m < r_idx を探す
  target   = neckline - (hv - neckline)

逆ヘッドアンドショルダー:
  neckline = maxima の中から l_idx < m < h_idx と h_idx < m < r_idx を探す → 平均
  target   = neckline + (neckline - hv)

トリプルトップ:
  neckline = min(close[b12 の最安値], close[b23 の最安値])
  target   = neckline - (peak_avg - neckline)

トリプルボトム:
  neckline = max(close[b12 の最高値], close[b23 の最高値])
  target   = neckline + (neckline - trough_avg)
```

### フラッグ系

```
強気フラッグ:
  pole_height = pole[-1] - pole[0]  (ポール終端 - ポール始端)
  breakout    = last_close
  target      = breakout + pole_height

弱気フラッグ:
  pole_height = pole[0] - pole[-1]  (ポール始端 - ポール終端)
  target      = breakout - pole_height
```

### 三角形 / ウェッジ系

```
共通:
  三角形の「最大幅」= 最初の山（または谷）の価格差 = close[maxima[-3]] - close[minima[-3]]
  収束点（x 軸）から target を決定。方向は direction で決まる。
  target = last_close ± max_width
```

### カップアンドハンドル

```
cup_depth_abs = left_mean - mid_mean  (絶対値)
target        = right_max + cup_depth_abs
```

## Confidence score calculation per pattern

| パターン | スコア計算式 | 根拠 |
|---|---|---|
| H&S / 逆H&S | `1.0 - shoulder_diff / 0.03` | 肩の差が小さいほど対称性が高く信頼度UP |
| ダブルトップ / ボトム | `1.0 - peak_diff / 0.02` | 2山の価格差が小さいほど信頼度UP |
| トリプルトップ / ボトム | `1.0 - max(diff12, diff23) / 0.03` | 3山のばらつき最大値が小さいほど信頼度UP |
| 上昇・下降三角形 | `1.0 - abs(flat_slope) / flat_thresh` | 水平側の傾きが 0 に近いほど信頼度UP |
| 対称三角形 / ウェッジ | `min(abs(slope_high), abs(slope_low)) / max(abs(slope_high), abs(slope_low))` | 両トレンドラインの傾きバランスが取れているほど信頼度UP |
| 強気 / 弱気フラッグ | `1.0 - rest_range / (pole_range * 0.5)` | 旗部分が小さいほど（ポールに対してタイト）信頼度UP |
| カップアンドハンドル | `1.0 - handle_dip / 0.05` | ハンドルの下落が小さいほど信頼度UP |

## Impact on existing tests

`PatternResult` に `confidence: float` を追加すると以下が破壊される：

1. **`tests/test_predictor.py:207-217` の `_make_pattern_result()`**
   - `PatternResult(name=..., direction=..., forecast_prices=..., forecast_index=...)` の呼び出しが `confidence` 欠落でエラー
   - `TestDrawPrediction` の全5テストが影響を受ける

2. **`detect_pattern()` の全 `PatternResult(...)` コンストラクト箇所（13箇所）**
   - 各コンストラクト呼び出しに `confidence=...` を追加する必要がある

3. **`draw_prediction()` のテスト**
   - `result.name` 表示テキストの期待値が `"ダブルトップ (信頼度: XX%)"` 形式になるか、従来の `"ダブルトップ"` のままかで確認が必要

**`dataclasses.field(default=0.5)` を使えば後方互換を保てる**が、デフォルト値が意味的に曖昧になるため、明示的に設定する方が望ましい。

## Risks/constraints discovered

- **H&S のネックライン計算が複雑**: 厳密なネックラインは「左肩の後の谷」と「頭の後の谷」の2点を結ぶ直線。簡易版として2谷の平均価格を使うことは可能だが、傾いたネックラインに対しては誤差が生じる
- **三角形・ウェッジの「最大幅」定義**: パターン開始点の山と谷の価格差とするか、全極値のレンジとするかで値が変わる。基準を固定しないとテストが不安定になる
- **`forecast_prices` の本数**: 現在は固定 10 本。目標価格がすでに現値に近い場合（浅いカップなど）、10本では収束・超過してしまう。本数を可変にするか上限を設けるか設計が必要
- **信頼度スコアのクリッピング**: 計算式がパターンによっては 0 未満や 1 超になりうる。`max(0.0, min(1.0, score))` でクランプ必須
- **`draw_prediction()` のテキスト変更**: 信頼度をパターン名に付加する場合、`TestDrawPrediction::test_draw_prediction_pattern_name_in_ax_text` の期待値 `"ダブルトップ"` が失敗する（パターン名テキストが変わるため）
