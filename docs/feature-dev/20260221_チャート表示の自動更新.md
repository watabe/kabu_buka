# チャート表示の自動更新

- **Date:** 2026-02-21

## Why Needed

チャートをみているだけで株価がわかる状態を作りたい。

現状、`--chart` フラグでチャートを表示しても、表示されるのは起動時点の1回きりのスナップショットであり、チャートを開いたままにしても株価は更新されない。リアルタイムに近い形で価格が変動する様子を確認するには、毎回 CLI を再実行する必要があり不便。自動更新によりチャートを開いたまま最新の株価を継続的に確認できる状態にする。

---

## Scope

### In-scope
- 一定間隔（例: 60秒）で yfinance から当日1分足データを再取得し、チャートを自動更新する
- チャートウィンドウを閉じるか Ctrl+C するまで更新を継続する
- `--interval` オプション（秒）で更新間隔を指定できるようにする（デフォルト: 60秒）
- 更新時は前のチャートを閉じて新しいチャートを描画する（または同一ウィンドウを更新する）
- 取引時間外（データ更新なし）でも再取得処理が走る（空データ時はエラーにせず継続）

### Out-of-scope
- WebSocket やストリーミング API を使ったミリ秒単位のリアルタイム更新
- 複数銘柄の同時表示・同時更新
- チャートウィンドウの常駐化（システムトレイへの格納など）
- 自動更新モードでの CUI テキスト出力の変更

---

## Success Criteria

- チャートウィンドウが指定インターバル（デフォルト60秒）ごとに自動更新される
- `--interval N` で更新間隔を変更できる
- Ctrl+C またはウィンドウを閉じると更新ループが終了しプログラムが正常終了する
- 取引時間外など再取得データが空でもクラッシュせず更新を継続する
- 既存の `--chart` なし起動（1回表示）の動作は変わらない

---

## Risks / Unknowns

- `mplfinance` / `matplotlib` のウィンドウ更新方法: `plt.pause()` + `plt.clf()` を使う方法と `FuncAnimation` を使う方法があり、どちらがより安定しているか調査が必要
- `mpf.show()` はブロッキング呼び出しのため、バックグラウンドスレッドでの更新ループとウィンドウ描画の競合が生じる可能性がある
- macOS 上の matplotlib バックエンド（TkAgg / MacOSX 等）によって非ブロッキング描画の挙動が異なる
- yfinance の1分足データは直近約7日間しか取れないが当日中の再取得は同一データの追記で問題ない
- 更新頻度が高すぎると yfinance へのリクエストが増え、レート制限に引っかかる可能性がある
