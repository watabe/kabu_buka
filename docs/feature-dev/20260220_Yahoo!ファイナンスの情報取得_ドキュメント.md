# Yahoo!ファイナンスの情報取得 - 機能ドキュメント

- **Date:** 2026-02-20

## Overview

ユーザーが銘柄コードを入力すると、`yfinance` ライブラリ経由でYahoo! Finance（`finance.yahoo.com`）から該当銘柄の株価情報を取得し、CLIの標準出力に整形表示するPythonツールを実装する。

個人利用・プロトタイプを主眼に置いた最初のバージョン（v1）として、シンプルなCLIアプリとして実装する。

---

## Goals / Non-Goals

### Goals
- 4桁の銘柄コード（東証銘柄）を入力として受け取る
- `yfinance` を使って銘柄の基本情報（銘柄名・株価・前日比・出来高など）を取得する
- 取得した情報をターミナル上に見やすく整形して表示する
- 存在しない銘柄コードや取得失敗時に適切なエラーメッセージを表示する

### Non-Goals
- リアルタイム自動更新（ティッカー表示）
- Webブラウザ向けUIの提供
- ポートフォリオ管理・銘柄情報の保存
- 複数銘柄の一括取得
- 商用・プロダクション環境での利用（ToS上の理由から）
- J-Quants API の利用（v1 では採用しない）

## Proposed Approach

### 技術選定

| 項目 | 選択 | 理由 |
|---|---|---|
| 実装言語 | Python 3.11+ | yfinance が Python ネイティブ。CLIツールとして最適 |
| データ取得 | `yfinance` (v1.2.0) | 公式APIが存在しないため現実的な選択肢。個人利用スコープに限定 |
| パッケージ管理 | `pip` + `requirements.txt` | シンプルな構成 |
| CLIインターフェース | `argparse` (標準ライブラリ) | 追加依存なし |

### ディレクトリ構成

```
prj-test/
├── src/
│   ├── __init__.py
│   ├── fetcher.py       # yfinance を使ったデータ取得
│   └── display.py       # 取得データの整形・表示
├── main.py              # CLIエントリポイント
├── requirements.txt
└── tests/
    ├── test_fetcher.py
    └── test_display.py
```

### 処理フロー

```
ユーザー入力 (例: 7203)
    ↓
main.py: argparse で銘柄コードを受け取る
    ↓
fetcher.py: "{銘柄コード}.T" に変換して yf.Ticker() を呼び出す
    ↓
（正常）StockInfo オブジェクトを返す
（エラー）例外をキャッチ → エラーメッセージを出力して終了
    ↓
display.py: StockInfo を整形してターミナルに出力
```

### 表示する情報（v1スコープ）

| フィールド | yfinance キー | 例 |
|---|---|---|
| 銘柄名 | `longName` | トヨタ自動車株式会社 |
| 銘柄コード | 入力値 | 7203 |
| 現在値 | `currentPrice` | ¥3,420 |
| 前日終値 | `previousClose` | ¥3,390 |
| 前日比 | 計算値 | +30 (+0.88%) |
| 始値 | `open` | ¥3,400 |
| 高値 | `dayHigh` | ¥3,435 |
| 安値 | `dayLow` | ¥3,398 |
| 出来高 | `volume` | 8,234,500 |
| 時価総額 | `marketCap` | ¥55.6兆 |

### CLI インターフェース

```bash
# 基本的な使い方
python main.py 7203

# 出力例
銘柄名    : トヨタ自動車株式会社 (7203.T)
現在値    : ¥3,420
前日比    : +30 (+0.88%)
始値/高値/安値: ¥3,400 / ¥3,435 / ¥3,398
出来高    : 8,234,500
時価総額  : ¥55.6兆
```

### エラーハンドリング方針

| ケース | 挙動 |
|---|---|
| 存在しない銘柄コード | "銘柄が見つかりませんでした: {code}" を出力、exit code 1 |
| ネットワークエラー | "データ取得に失敗しました。通信状況を確認してください。" を出力、exit code 1 |
| タイムアウト | 同上（10秒タイムアウトを設定） |
| 取引時間外 | 取得できたデータをそのまま表示（前日終値など） |

## API or Interface Changes

新規実装のため、既存APIへの変更はなし。

外部依存として `yfinance` の非公式内部エンドポイントを使用する。エンドポイント仕様は公開されておらず、変更された場合はライブラリアップデートで追従する。

---

## Data Model Changes

永続化・DB使用はなし。取得データは実行時にメモリ上で保持するのみ。

内部データ構造（`StockInfo` dataclass 想定）:

```python
@dataclass
class StockInfo:
    code: str            # 入力銘柄コード (例: "7203")
    ticker: str          # yfinance ティッカー (例: "7203.T")
    name: str            # 銘柄名
    current_price: float
    previous_close: float
    open_price: float
    day_high: float
    day_low: float
    volume: int
    market_cap: int | None
```

## Test Plan Outline

### 単体テスト（`tests/test_fetcher.py`）
- `yf.Ticker` をモック化して `fetch_stock_info()` が正常な `StockInfo` を返すことを検証
- 存在しない銘柄コードのとき適切な例外が発生することを検証
- ネットワークエラー時に例外が伝播することを検証
- `.T` サフィックスが自動付与されることを検証

### 単体テスト（`tests/test_display.py`）
- `StockInfo` を与えたとき期待する文字列が出力されることを検証
- 前日比の正負・ゼロの各ケースで表示が正しいことを検証
- `market_cap` が `None` のときクラッシュしないことを検証

### 統合テスト・手動テスト
- 実際の銘柄コード（例: `7203`、`6758`、`9984`）で正常動作を確認
- 存在しないコード（例: `9999`）でエラーメッセージが表示されることを確認
- ネットワーク切断時のエラーハンドリングを確認
