# チャート表示時の文字化け解消 - 機能ドキュメント

- **Date:** 2026-02-21

## Overview

`src/chart.py` の `plot_intraday()` 内で `mplfinance` にスタイルを渡す際、日本語対応フォントを自動検出して `mpf.make_mpf_style(rc=...)` 経由で注入する。これにより、チャートのタイトル・軸ラベルの日本語文字化けを解消する。

変更は `src/chart.py` 1ファイルのみで完結し、他のモジュールへの影響はない。

---

## Goals / Non-Goals

### Goals
- `mpf.plot()` に渡すスタイルに日本語フォントを自動的に組み込み、チャートタイトル・軸ラベルの日本語を正しく表示する
- 利用可能な日本語フォントを優先リストで自動検出し、ユーザーが手動設定不要にする
- 日本語フォントが見つからない環境（CI 等）でも `axes.unicode_minus = False` を適用しマイナス記号の文字化けを防ぐ
- フォント未検出時でもクラッシュせず、豆腐表示のまま動作継続する

### Non-Goals
- Windows・Linux 向けの日本語フォントインストールや設定手順の提供
- グラフのスタイル・配色変更
- CLI テキスト出力（ターミナル）の文字化け対応
- フォントキャッシュの自動再構築

---

## Proposed Approach

### 変更箇所: `src/chart.py` のみ

#### 追加するプライベート定数・関数

```python
_JP_FONT_CANDIDATES = [
    "Hiragino Sans",           # macOS 標準（全バージョン）
    "Hiragino Kaku Gothic Pro",
    "Hiragino Maru Gothic Pro",
    "Arial Unicode MS",        # macOS + MS Office
    "Noto Sans CJK JP",        # 手動インストール / Linux
    "IPAGothic",
    "IPAexGothic",
    "Yu Gothic",               # Windows
    "Meiryo",                  # Windows
]

def _resolve_japanese_font() -> str | None:
    """利用可能な日本語フォントを優先リストから最初に見つけたものを返す。"""
    available = {f.name for f in fm.fontManager.ttflist}
    return next((f for f in _JP_FONT_CANDIDATES if f in available), None)
```

#### `plot_intraday()` の変更

`style="yahoo"` の文字列渡しを廃止し、`mpf.make_mpf_style()` で生成したスタイルオブジェクトに置き換える。

```python
def plot_intraday(df: pd.DataFrame, ticker: str) -> None:
    if df.empty:
        raise ValueError(f"表示するデータがありません: {ticker}")

    plot_df = df[["Open", "High", "Low", "Close", "Volume"]].copy()

    font = _resolve_japanese_font()
    rc: dict = {"axes.unicode_minus": False}
    if font:
        rc["font.family"] = font

    style = mpf.make_mpf_style(base_mpf_style="yahoo", rc=rc)

    mpf.plot(
        plot_df,
        type="candle",
        style=style,           # 文字列 "yahoo" から style オブジェクトに変更
        title=f"{ticker}  1日 (1分足)",
        ylabel="株価 (円)",
        ylabel_lower="出来高",
        volume=True,
        figsize=(14, 7),
        show_nontrading=False,
    )
```

### なぜ `make_mpf_style(rc=...)` を使うか

`mpf.plot(style="yahoo")` は内部でスタイルシートを適用する。将来のバージョン変更でベーススタイルが変わっても、`make_mpf_style(rc=...)` の `rc` はスタイル適用後に上書きされるため常に有効。

### 変更の最小性

- 追加インポート: `matplotlib.font_manager as fm`（`matplotlib` は mplfinance 経由で既にロード済みのため追加コストなし）
- 既存のテスト (`tests/test_chart.py`) は `mpf.plot` をモック済みのため、`make_mpf_style` を追加モックする形でテストを拡張する
