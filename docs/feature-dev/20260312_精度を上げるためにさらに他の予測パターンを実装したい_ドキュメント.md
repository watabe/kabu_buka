## Proposed Approach

### 新規ヘルパー関数

#### `_fit_trendline(indices, values) -> tuple[float, float]`

三角形・ウェッジ系パターンの回帰直線計算に使う共通ヘルパー。`numpy.polyfit` で 1 次回帰し、傾き (slope) と切片 (intercept) を返す。

```python
def _fit_trendline(indices: np.ndarray, values: np.ndarray) -> tuple[float, float]:
    """1次線形回帰で傾きと切片を返す。"""
    slope, intercept = np.polyfit(indices, values, 1)
    return float(slope), float(intercept)
```

#### `_pct_range(values: np.ndarray) -> float`

フラッグのポール検出・カップ検出に使う価格レンジの百分率計算。

```python
def _pct_range(values: np.ndarray) -> float:
    """(max - min) / min を返す。values が空の場合は 0.0。"""
    if len(values) == 0:
        return 0.0
    mn = values.min()
    if mn == 0:
        return 0.0
    return float((values.max() - mn) / mn)
```

### 各パターンの検出ロジック詳細

#### トリプルトップ

```
条件:
- 直近の極大が 3 つ以上存在する
- 末尾 3 つの極大 (p1, p2, p3) の価格差がすべて ≤ 3%
  abs(v1 - v2) / max(v1, v2) <= 0.03
  abs(v2 - v3) / max(v2, v3) <= 0.03
- p1〜p2 の間と p2〜p3 の間にそれぞれ極小が 1 つ以上存在する
- ヘッドアンドショルダー条件（中央が最大）を満たさないこと
```

#### トリプルボトム

```
条件:
- 直近の極小が 3 つ以上存在する
- 末尾 3 つの極小 (t1, t2, t3) の価格差がすべて ≤ 3%
- t1〜t2 の間と t2〜t3 の間にそれぞれ極大が 1 つ以上存在する
- 逆ヘッドアンドショルダー条件（中央が最小）を満たさないこと
```

#### 上昇三角形

```
条件:
- 直近の極大が 3 つ以上存在する → _fit_trendline で回帰
  - 極大の傾き (slope_high) が -0.001 ≤ slope_high ≤ 0.001（ほぼ水平）
- 直近の極小が 3 つ以上存在する → _fit_trendline で回帰
  - 極小の傾き (slope_low) が > 0.001（上昇）
```

#### 下降三角形

```
条件:
- 極大の傾き (slope_high) が < -0.001（下降）
- 極小の傾き (slope_low) が -0.001 ≤ slope_low ≤ 0.001（ほぼ水平）
```

#### 対称三角形

```
条件:
- 極大の傾き (slope_high) が < -0.001（下降）
- 極小の傾き (slope_low) が > 0.001（上昇）
- 直前 20 本の Close の線形傾きで方向を決定
```

#### ライジングウェッジ

```
条件:
- slope_high > 0.001 かつ slope_low > 0.001（両方上昇）
- slope_low > slope_high（支持線の上昇が抵抗線より急 = 収束）
- direction = "down"（上昇後の下落転換）
```

#### フォーリングウェッジ

```
条件:
- slope_high < -0.001 かつ slope_low < -0.001（両方下降）
- slope_high < slope_low（抵抗線の下落が支持線より急 = 収束）
- direction = "up"（下落後の上昇転換）
```

#### 強気フラッグ（Bullish Flag）

```
条件:
- 全データの前半 N 本（pole_len = len(close) // 3）にポール候補を探す
  pct_range(close[i : i + pole_len]) >= 0.03 かつ close が上昇傾向
- ポール後の後半部分（残りのデータ）が小さいチャンネル内に収まる
  pct_range(close[pole_end:]) < pole_range * 0.5
- direction = "up"
```

#### 弱気フラッグ（Bearish Flag）

```
条件:
- ポール候補で急落があること（close が下降傾向）
- その後の後半部分が小チャンネル内に収まる
- direction = "down"
```

#### カップアンドハンドル

```
条件:
- データを 3 分割: 左端(L)・中央(C)・右端(R)
  L = close[:n//3], C = close[n//3 : 2*n//3], R = close[2*n//3:]
- カップ: mean(L) > mean(C) かつ mean(R) > mean(C)（U字）
  (mean(L) - min(C)) / mean(L) >= 0.03（カップの深さ ≥ 3%）
- ハンドル: R の最後の 1/4 が R の最高値から ≤ 5% 下落した後に回復している
  max(R) - close[-1] の比率 ≤ 0.05
- direction = "up"
```
