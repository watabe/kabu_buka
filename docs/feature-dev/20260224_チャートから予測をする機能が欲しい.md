# チャートから予測をする機能が欲しい

- **Date:** 2026-02-24

## Why Needed

ON/OFFをすることでチャート上の表示を切り替えて現状グラフに対してオーバーラップする形で予測が表示されるようにしたい。また株価予測についてはグラフから見るパターンがいくつかあるのでそのパターンのうちどのパターンでの予測なのかも表示されるようにしたい。

現在のチャートは実績データを表示するだけに留まっており、次の値動きを視覚的に判断するための補助情報がない。テクニカル分析で一般的に使われるチャートパターン（例: ダブルトップ、ヘッドアンドショルダー、フラグ、トライアングルなど）を自動検出し、検出されたパターンに基づく将来予測ラインをチャートにオーバーラップ表示することで、ユーザーが相場の方向性をチャートから一目で把握できるようにしたい。

---

## Scope

### In-scope

- `--predict` フラグ（ON/OFF トグル）を CLI に追加し、指定時のみ予測オーバーレイを表示する
- 1分足の当日データをもとに、以下のテクニカルパターンを検出する
  - ダブルトップ / ダブルボトム（反転シグナル）
  - ヘッドアンドショルダー / 逆ヘッドアンドショルダー（反転シグナル）
  - 上昇フラグ / 下降フラグ（継続シグナル）
  - 上昇トライアングル / 下降トライアングル / 対称トライアングル（継続・反転）
- 検出されたパターン名をチャート上または上部情報パネルに表示する
- パターンが検出された場合、チャートの将来時間帯に予測ラインをオーバーラップ描画する（点線または半透明のライン）
- `plot_intraday()` と `plot_intraday_live()` の両方で `--predict` フラグを尊重する
- パターンが未検出の場合は「パターン未検出」と表示してオーバーレイを描画しない

### Out-of-scope

- 機械学習・ディープラーニングモデルによる価格予測（ルールベースのパターン検出のみ）
- 日足・週足など1分足以外の時間軸への対応
- 複数パターンの同時検出・描画（最も強いシグナルの1パターンのみ）
- バックテスト・予測精度の評価機能
- パターン検出パラメータのユーザー設定機能

---

## Success Criteria

- `python main.py 7203 --chart --predict` を実行するとチャートに予測オーバーレイが表示される
- `--predict` なしの場合は従来通りの表示で変化なし
- 検出されたパターン名（例: "ダブルトップ"）がチャート上に表示される
- 予測ラインがローソク足チャートの時間軸の延長線上にオーバーラップ描画される
- パターン未検出時は「パターン未検出」と表示され、予測ラインは描画されない
- `plot_intraday_live()` でも自動更新のたびに予測が再計算・再描画される
- 既存の 79 件のテストが引き続き PASS する

## Risks/Unknowns

- 1分足の当日データは取引時間内であれば十分な本数（数十〜数百本）があるが、取引時間外や取引開始直後は本数が少なくパターン検出が困難な場合がある
- ルールベースのパターン検出は閾値・ウィンドウサイズの設定により結果が大きく変わる。最適な閾値の決定が必要
- mplfinance の External Axes Mode（`plot_intraday_live()` の `animate()` 内）でオーバーレイラインを毎フレーム再描画する際のパフォーマンス
- `axes[0]` に `matplotlib.lines.Line2D` などを追加してオーバーレイ描画した場合、`ax.clear()` で消えてしまうため `animate()` 内で毎回再描画が必要
- 予測の「将来時間」を1分足の DatetimeIndex から外挿するロジックの実装複雑度（取引時間の考慮が必要）
- 検出ロジックの精度・誤検知リスク（誤検知でも「予測の参考情報」として明示し過信を防ぐ表示が必要）
