## Findings Summary

### 実証済みの実装パターン（External Axes Mode + FuncAnimation）

```python
import matplotlib.animation as animation

# Step 1: returnfig=True で fig/axes を取得（初期描画）
fig, axes = mpf.plot(df, type='candle', style=style, volume=True,
                     figsize=(14, 7), returnfig=True)
ax1 = axes[0]   # ローソク足パネル
ax2 = axes[2]   # 出来高パネル

# Step 2: 一度だけ余白確保
fig.subplots_adjust(bottom=0.12)

# Step 3: 情報テキスト領域を初期化
info_text_obj = fig.text(0.01, 0.02, '', fontsize=9, va='bottom')

# Step 4: フレームコールバック
def animate(ival):
    df_new = fetch_intraday_data(code)   # yfinance 再取得
    if df_new.empty:
        return                           # 空データはスキップ（クラッシュしない）
    ax1.clear()
    ax2.clear()
    mpf.plot(df_new, ax=ax1, volume=ax2, type='candle', style=style)
    # fig.text は ax.clear() 後も残るので remove して更新
    info_text_obj.set_text(build_info_text(df_new, info))

# Step 5: FuncAnimation（interval はミリ秒）
ani = animation.FuncAnimation(fig, animate,
                               interval=interval_sec * 1000,
                               cache_frame_data=False)
mpf.show()
```

### 確認済み事項（実環境 Agg バックエンド）

| 確認内容 | 結果 |
|---|---|
| `mpf.plot(returnfig=True)` → `(fig, axes)` 取得 | OK |
| `axes[0]`=price, `axes[2]`=volume（axes は4要素） | OK |
| `mpf.plot(ax=ax1, volume=ax2, ...)` で外部軸に再描画 | OK |
| `ax1.clear()` / `ax2.clear()` 後も `fig.texts` は残存 | OK |
| `fig.text` オブジェクトの `set_text()` で in-place 更新が可能 | OK |
| `FuncAnimation(cache_frame_data=False)` で warnings なし | OK |
| `FuncAnimation` の `interval` はミリ秒単位 | OK（60000 = 60秒）|

### `fig.text` 更新の2種類の方法

**方法A**: `set_text()` でオブジェクトを再利用（推奨）
```python
txt = fig.text(0.01, 0.02, 'initial', fontsize=9, va='bottom')
# フレームごとに:
txt.set_text('updated text')
```

**方法B**: `remove()` して再追加
```python
txt.remove()
txt = fig.text(0.01, 0.02, 'updated text', fontsize=9, va='bottom')
```
方法Aの方がオブジェクト管理が単純なため採用。

### `--interval` なし（1回表示）との共存

既存の `plot_intraday(df, info)` 関数は変更せず維持し、`plot_intraday_live(code, info, interval_sec)` を別関数として追加する。`main.py` で `--interval` 指定の有無に応じて呼び分ける。

## Risks / Constraints Discovered

- **`mpf.show()` はブロッキング呼び出し**だが、`FuncAnimation` を作成してから `mpf.show()` を呼ぶことでアニメーションループとウィンドウ表示が一体で動作する（公式 demo2.py と同じパターン）。
- **GUI バックエンド依存**: Agg バックエンド（ヘッドレス）では `mpf.show()` は何もしない。実際の表示には TkAgg / MacOSX 等の GUI バックエンドが必要。これは既存の `plot_intraday` と同じ制約。
- **`FuncAnimation` の `ani` 変数のスコープ**: `ani` をローカル変数にすると GC に回収されアニメーションが止まる（matplotlib の公式 warning に記載）。`ani` は外部スコープで保持する必要がある。
- **テスト**: `FuncAnimation` を含む関数のテストでは `mpf.show` だけでなく `FuncAnimation` もモック化する必要がある。
- **`fetch_intraday_data` の依存**: `chart.py` が `fetcher.py` をインポートすることになるが、`fetcher.py` は `chart.py` をインポートしないため循環依存は生じない。
- **yfinance のレート制限**: 60秒ごとの再取得は合理的だが、それより短い間隔は避けること。

## Open Questions

- **`plot_intraday_live` に `code` を渡すか、`fetch_fn: Callable` を渡すか？** `code` を渡す設計（`fetcher.py` を内部 import）の方がシンプルだが、テスト時に `fetch_intraday_data` をモックする必要がある。`fetch_fn` を引数にする方がテスタブルだが呼び出し側が少し複雑になる。
- **取引時間外の空データ時の挙動**: スキップ（現状データを表示し続ける）か、ウィンドウにメッセージを表示するか？
- **`--interval` なしで `--chart` を指定したとき**: 現行の `plot_intraday`（1回表示）を維持するか、デフォルトで60秒更新を有効にするか？
- **ウィンドウを閉じたとき**: `FuncAnimation` の `event_source` を止める処理が必要か、`mpf.show()` が返るだけで十分か？
