## Root cause analysis

### 問題1（主因候補）: `ani` のガベージコレクション

`ani` はローカル変数として保持されているが、CPython の参照カウントでは `mpf.show()` がブロックしている間はスコープ内に留まる。しかし `mpf.show()` が **非ブロッキングで返る**（インタラクティブバックエンド等）場合、関数が終了して `ani` が即座に GC され、アニメーションタイマーが停止する。

> `matplotlib.animation` ドキュメント: "The animation object must be kept alive for the duration of the animation."
> コード内のコメント `# noqa: F841  (ani はスコープ保持が必要)` がこのリスクを認識していることを示すが、非ブロッキング環境では不十分。

### 問題2（副因）: `mpf.show()` とイベントループの非互換

`mpf.show()` は内部で `plt.show()` を呼ぶが、matplotlib バックエンドが interactive モードの場合（`plt.ion()` が呼ばれていた場合など）、`plt.show()` は即時返却する。その結果 `ani` がスコープを外れ、FuncAnimation タイマーが破棄される。

### 問題3: `animate()` が `fig.canvas.draw_idle()` を呼ばない

`blit=False`（デフォルト）の `FuncAnimation` は内部で `fig.canvas.draw()` を呼ぶ設計だが、`ax.clear()` + `mpf.plot(ax=ax1, volume=ax2, ...)` という重い再描画パターンとの組み合わせにより、一部バックエンドで再描画がトリガーされない可能性がある。

### 問題4: `info` が初期値のまま更新されない

`animate()` は closure で `info`（初期 `StockInfo`）を参照する。`fetch_intraday_data()` は OHLCV データのみを返すため、`info.current_price` / `info.volume` は初回取得時の値のまま。グラフ上部の現在値・出来高テキストが更新されない。

## Risks/constraints discovered

- **テストがモックのみ**: `FuncAnimation` を全てモックしているため、実際のタイマー発火・GUI イベントループとの協調は未検証
- **バックエンド依存**: TkAgg / macOS Quartz / Qt など GUI バックエンドごとに `plt.show()` のブロッキング挙動が異なる
- **`ax.clear()` の副作用**: mplfinance が初期描画時に設定した xlabel/ylabel/tick 書式がクリアされ、`mpf.plot(ax=ax1, ...)` で再設定されない可能性がある
- **`fetch_intraday_data()` の所要時間**: ネットワーク遅延が `interval_sec` を超えると animate コールバックが前のコールと重複する（FuncAnimation はそれを考慮しない）
- **予測計算コスト**: `detect_pattern()` は全データの極値探索を毎フレーム行うため、データ本数が多い（300本超）場合にパフォーマンス劣化の可能性がある
