## Approach

### `fig.suptitle()` を使う（推奨）

mplfinance で `returnfig=True` を使うと `(fig, axes)` が返る。`fig.suptitle(text)` はフィギュア全体の**最上部**にタイトルを追加し、`Text` オブジェクトを返す。

```python
suptitle_txt = fig.suptitle(_build_info_text(plot_df, info), fontsize=11, y=0.98)
```

**重要な挙動確認:**

- `ax.clear()` は axes 内のアーティストを削除するが、`fig.suptitle()` で追加した `Text` は**フィギュアレベル**に属するため消えない
- → `plot_intraday_live()` の `animate()` コールバック内で `suptitle_txt.set_text(new_text)` を呼べばインプレース更新できる（既存の `info_txt.set_text()` パターンと同じ）
- `fig.subplots_adjust(top=0.88)` で上部マージンを確保する必要がある（既存の `bottom=0.12` は削除可）

## Risks/Constraints Discovered

1. **`TestPlotIntradayInfoText` 7件が破壊的変更になる**: `fig.text()` → `fig.suptitle()` への変更で、`call_args[0][2]`（第3引数）ではなく `call_args[0][0]`（第1引数）を参照するよう変更が必要（`suptitle` は第1引数がテキスト文字列）
2. **`fig.subplots_adjust` の上下マージン**: 既存の `bottom=0.12` は下部テキストのための余白だったが、上部テキストに移行する場合 `top=0.88` に変更する。ただし `bottom` も残すと上下で余白が広くなる可能性があり、レイアウトのバランス調整が必要
3. **`y=` パラメータ調整**: `fig.suptitle(y=0.98)` で十分な余白が取れるか、実環境で確認が必要（テストではモック化するため問題なし）
4. **`plot_intraday_live()` の `animate()` 内タイトル**: `mpf.plot(ax=ax1, ...)` は External Axes Mode のため `title=` を渡さない。`suptitle_txt.set_text()` で更新する方針は継続可能

## Open Questions

1. 下部 `fig.text()` を**削除して上部 `fig.suptitle()` のみにする**か、**上下両方に表示**するかの選択 → 要件から「上部に移動して一目でわかるようにする」と解釈し**削除して上部のみ**が適切と判断
2. `fig.subplots_adjust(bottom=0.12)` を削除するか継続するか → 下部テキストがなくなるなら不要。削除してよい
3. `fig.suptitle` の `fontsize` は現行の下部テキスト `fontsize=9` より大きく（例: `fontsize=11`）すべきか → 視認性向上の観点から `11` または `10` が適当
