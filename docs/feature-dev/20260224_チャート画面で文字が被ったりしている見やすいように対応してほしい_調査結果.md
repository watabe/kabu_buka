## Relevant Code Areas

### 衝突の根本原因

**`src/chart.py` L123–141（`plot_intraday()`）および L171–191（`plot_intraday_live()`）:**

```python
# mplfinance への title= パラメータ（L127 / L175）
fig, _ = mpf.plot(
    ...,
    title=f"{info.ticker}  1日 (1分足)",   # ← 問題の根本
    ...
)

# 直後の fig.text（L136–141）
fig.subplots_adjust(top=0.82)
fig.text(0.01, 0.975, parts["name"], ...)   # ← ここと衝突
fig.text(0.42, 0.975, parts["price"], ...)
fig.text(0.58, 0.975, parts["change"], ...)
fig.text(0.01, 0.930, parts["meta"], ...)
```

**衝突のメカニズム:**

mplfinance は `title=` パラメータを内部で `fig.suptitle(title)` として描画する（volume=True で複数パネルがある場合に図全体のタイトルとして扱う）。matplotlib の `fig.suptitle()` デフォルト y 位置は **y=0.98** であり、直後に呼んでいる `fig.subplots_adjust(top=0.82)` はサブプロットの配置のみ変更し `suptitle` のy座標には影響しない。したがって:

- mplfinance `suptitle`: y ≈ 0.98（`"{ticker}  1日 (1分足)"` が描画される）
- 我々の `fig.text` Row 1: y = 0.975（銘柄名・現在値・前日比が描画される）

**→ y=0.975 と y=0.98 の差はわずか 0.005（7インチ高で 0.035インチ ≈ 2.5pt）しかなく、両者が完全に重なる。**

### 追加で確認した問題点

1. **ticker の二重表示**: `parts["name"]` は `"{info.name} ({info.ticker})"` を含み、mpf title も同じ ticker を含む → 冗長かつ重複
2. **Row 1 3要素が同一 y 座標**: 銘柄名（x=0.01）・現在値（x=0.42）・前日比（x=0.58）が y=0.975 に並ぶため、将来的な銘柄名の長さ増加で水平衝突リスクがある
3. **`ax.set_title()` の誤解**: 旧設計では「mpf title は axes レベルの `ax.set_title()` なので y≈0.82 付近にある」と推定していたが、実際は `fig.suptitle()` のため y≈0.98

## Approach

### 確定した修正方針

1. **`mpf.plot()` から `title=` を削除**（または空文字 `title=""`）し mplfinance の `suptitle` 描画を抑制する
2. 代わりに **3行レイアウト**を採用してすべての情報を被りなく表示する
3. 「1日 (1分足)」の期間ラベルは Row 1 に統合する

### 新レイアウト設計

```
[figure top = 1.0]
y=0.960  Row 1: {info.name} ({info.ticker})  1日 (1分足)   13pt bold  左寄せ
y=0.922  Row 2: ¥{price} 16pt bold  |  {change} 12pt green/red
y=0.885  Row 3: 出来高: xxx  時価総額: xxx  取得: HH:MM   9pt gray
[subplots top = 0.85]
[chart area]
```

`fig.subplots_adjust(top=0.85)` で3行分の余白を確保（現行の 0.82 より若干広げる）。

Row 2 の price と change は横並びだが、長い銘柄名がある Row 1 とは行を分けるため水平衝突リスクがゼロになる。

### `animate()` の更新対象

| テキストオブジェクト | animate 内での更新 |
|---|---|
| `_name_txt`（Row 1） | 更新なし（銘柄名・コード・期間は変化しない） |
| `_price_txt`（Row 2） | `set_text()` |
| `_change_txt`（Row 2） | `set_text()` + `set_color()` |
| `_meta_txt`（Row 3） | `set_text()` |

## Risks/Constraints Discovered

1. **`mpf.plot(title="")` で suptitle が空文字で残る場合**: `title=` を `None` にするか `title=""` を渡すかで挙動が異なる可能性がある。mplfinance のバージョンによっては `title=None` が必要。調査結果として `title=""` を指定し検証することを推奨。
2. **`subplots_adjust(top=0.85)` の微調整**: 0.82 → 0.85 への変更で上部余白が広がるが、3行分のテキストに十分なスペースが確保できるか実環境確認が必要（テストはモックのため問題なし）。
3. **Row 1 の `_name_txt` が `animate()` 内で更新されないことのテスト**: `animate()` では `_price_txt` / `_change_txt` / `_meta_txt` のみ更新する設計を明示的にテストする必要がある。
4. **既存 `TestPlotIntradayInfoText` および `TestPlotIntradayLive` への影響**: `title=` 削除・y 座標変更・行数変更により、テストで検証している `call_args` の内容が変わる。変更テストを最小限に留めつつ新レイアウトを正確に検証する方針をとる。

## Open Questions

1. `mpf.plot(title="")` と `title=None` のどちらが mplfinance で suptitle を完全に非表示にするか → ドキュメントでは `title=""` を採用し、動作確認で問題があれば `None` に変更
2. Row 2 で price と change を同一行に並べる場合の x 座標: `_price_txt` を x=0.01、`_change_txt` を x=0.20 程度に設定するのが自然か。"¥6,000" が最長で約5文字 × 16pt ≈ 0.09 fig coords。0.20 から change 開始なら十分なマージンがとれる
3. `_build_info_parts()` の `"name"` フィールドに期間ラベル `"1日 (1分足)"` を含めるか、呼び出し側で結合するか → `_build_info_parts()` には含めず、`plot_intraday()` / `plot_intraday_live()` 呼び出し側でラベルを付加する（live モードは `"自動更新"` も付加できる）
