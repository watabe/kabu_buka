## Findings Summary

### 根本原因

matplotlib のデフォルトフォント `DejaVu Sans` は CJK（中国語・日本語・韓国語）グリフを含まない。日本語文字コードポイントを描画しようとすると、グリフが見つからず □（豆腐）として表示される。警告ログ例:
```
Glyph 26666 (CJK UNIFIED IDEOGRAPH) missing from current font.
```

### 確認済みの環境情報

| 項目 | 値 |
|---|---|
| matplotlib バージョン | 3.10.8 |
| キャッシュ/設定ディレクトリ | `~/.matplotlib/` |
| 現在の `font.family` | `['sans-serif']` |
| 現在の `font.sans-serif` 先頭 | `DejaVu Sans` |

### このシステムで利用可能な日本語フォント（確認済み）

- `Hiragino Sans`（W0〜W9 全10ウェイト、`/System/Library/Fonts/ヒラギノ角ゴシック W*.ttc`）
- `Hiragino Maru Gothic Pro`
- `Hiragino Mincho ProN`
- `Osaka`
- `Toppan Bunkyu Gothic`

→ **`Hiragino Sans`** がすべての macOS に標準搭載されており、確実に利用できる。

### mplfinance のフォント上書き問題

mplfinance の `style="yahoo"` は内部で `fast` ベーススタイルを使用する（seaborn ではない）。`fast` スタイルは `font.family` を上書きしないため、**`style="yahoo"` を使う場合は `rcParams` のグローバル設定のみで動作する可能性がある**。

ただし確実を期すため、`mpf.make_mpf_style(base_mpf_style="yahoo", rc={"font.family": ..., "axes.unicode_minus": False})` でフォントを明示的に注入する方法を採用する。これはスタイル適用後に `rc` を上書きするため常に有効。

### 修正パターン（確定）

```python
import matplotlib
import matplotlib.font_manager as fm
import mplfinance as mpf

_JP_FONT_CANDIDATES = [
    "Hiragino Sans",
    "Hiragino Kaku Gothic Pro",
    "Arial Unicode MS",
    "Noto Sans CJK JP",
    "IPAGothic",
    "Yu Gothic",
    "Meiryo",
]

def _resolve_japanese_font() -> str | None:
    available = {f.name for f in fm.fontManager.ttflist}
    return next((f for f in _JP_FONT_CANDIDATES if f in available), None)

def plot_intraday(df, ticker):
    font = _resolve_japanese_font()
    rc = {"axes.unicode_minus": False}
    if font:
        rc["font.family"] = font

    style = mpf.make_mpf_style(base_mpf_style="yahoo", rc=rc)

    mpf.plot(df[["Open","High","Low","Close","Volume"]],
             type="candle", style=style,
             title=f"{ticker}  1日 (1分足)",
             ylabel="株価 (円)", ylabel_lower="出来高",
             volume=True, figsize=(14, 7), show_nontrading=False)
```

### フォントキャッシュについて

- `Hiragino Sans` は macOS 標準搭載フォントのため、すでにキャッシュに登録済み
- 新規フォントをインストールした場合は `~/.matplotlib/fontlist-*.json` を削除してキャッシュ再構築が必要

### `axes.unicode_minus = False` について

ユニコードのマイナス記号（U+2212）も DejaVu Sans に含まれないためボックスになる。日本語フォントが見つからない環境でも、この設定は常に有効化すべき。

## Risks / Constraints Discovered

- **`make_mpf_style` の `rc` キーは matplotlib rcParams キー名と完全一致が必要。** タイポで警告なしに無視されることがある。
- **`yahoo` スタイルの内部実装変更リスク:** mplfinance の将来バージョンで `yahoo` スタイルのベーススタイルが変わると、`rcParams` グローバル設定のみのアプローチは再び機能しなくなる可能性がある。`make_mpf_style(rc=...)` を使うことでこのリスクを回避する。
- **Linux/CI 環境:** `Hiragino Sans` は macOS 専用。CI 環境ではフォント候補がすべて見つからない場合がある。`axes.unicode_minus = False` は常に設定されるため最低限の文字化け（マイナス記号）は防げるが、日本語はフォールバック（英語タイトル変更 or 空白）になる。
- **テストへの影響:** `test_chart.py` では `mpf.plot` をモック済みのため `make_mpf_style` の呼び出し検証は現在のテストではカバーされていない。`make_mpf_style` の引数検証テストを追加することが望ましい。
