# チャート表示時の文字化け解消

- **Date:** 2026-02-20

## Why Needed

チャートをGUIで表示した際に日本語が文字化けしてしまうので解消したい

## Scope

### In-scope
- `mplfinance` で表示するチャートのタイトル・軸ラベルの日本語を正しく表示する
- macOS 環境での日本語フォント設定を自動化する
- フォントが見つからない場合のフォールバック処理を実装する

### Out-of-scope
- Windows・Linux 環境での日本語フォント対応（macOS を優先とする）
- グラフのスタイル・デザイン変更
- CLI のテキスト出力（ターミナル表示）の文字化け対応

## Success Criteria

- チャートウィンドウのタイトルに日本語（例: `7203.T  1日 (1分足)`）が正しく表示される
- 株価軸ラベル「株価 (円)」・出来高軸ラベル「出来高」が化けずに表示される
- フォント設定がコード内で自動的に行われ、ユーザーが手動設定不要である
- 日本語フォントが存在しない環境でも英語表示にフォールバックしてクラッシュしない

## Risks / Unknowns

- macOS にインストール済みの日本語フォント名が環境によって異なる可能性がある（`Hiragino Sans` / `Hiragino Kaku Gothic Pro` など）
- `matplotlib` のフォントキャッシュを削除しないと設定が反映されない場合がある
- `mplfinance` が内部で `matplotlib` のフォント設定を上書きする可能性がある
- CI/CD 環境や Docker コンテナには日本語フォントが存在しないことが多い
